07 — Technical deep dive teaser
KAIROS
@withkairos
Engineering
We moved from
JSONB to an
append-only log.
## query time at p95 JSONB (before): 340ms event log (after): 18ms ## why it matters traces are write-once, read-many schema: events.sql → github
withkairos.dev/engineering 2026-05-30
Tweet copy
We rebuilt trace storage from JSONB to an append-only event log. Query time dropped from 340ms to 18ms at p95. The insight: traces are write-once, read-many. JSONB's flexibility is wasted on an immutable record. An ordered event log is a better fit structurally and faster in practice. Schema is on GitHub.
